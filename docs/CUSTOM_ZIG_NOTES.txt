Bu dosya, projede standart Zig stdlib veya tipik Zig iskeleti disinda bizim elle ekledigimiz yapilari aciklar.

1. Vendored Zig toolchain kullanimi
- `build.zig` icinde `.zig_lib_dir = b.path("vendor/zig-std")` kullaniliyor.
- Repo icinde `vendor/zig/zig-real` ve `vendor/zig-std/` bulunuyor.
- `tools/zig-dispatcher.c` ile repo icinde `zig` cagrisi local toolchain'e yonlendiriliyor.
- Amac: sistem Zig'i degisse bile repo icinde sabit toolchain davranisi saglamak.

2. Linux TCP SYN profile resolver
- `src/network_core.zig` icinde Linux SYN `window` ve `wscale` alanlari artik canli `TCP_INFO` socket telemetry ile degil, kernelin `tcp_select_initial_window()` mantigina gore `/proc/sys` degerlerinden turetiliyor.
- Kaynak referanslari:
  - `linux/net/ipv4/tcp_output.c` — `tcp_select_initial_window`, SYN `th->window = min(tp->rcv_wnd, 65535U)`
  - `include/net/tcp.h` — `__tcp_win_from_space`, `tcp_full_space`, `TCP_DEFAULT_SCALING_RATIO`, `MAX_TCP_WINDOW`, `TCP_MAX_WSCALE`
  - `Documentation/networking/ip-sysctl.rst` — `tcp_rmem`, `tcp_window_scaling`
- Amac: Paket serializer'i canli socket acma/yetki ihtiyacindan ayirmak ve gerçek kernel default'larina baglamak.

3. TCP stack window profile helper'lari
- `loadLinuxTcpSysctlProfile()` `/proc/sys/net/ipv4/tcp_rmem`, `/proc/sys/net/ipv4/tcp_window_scaling`, `/proc/sys/net/ipv4/tcp_workaround_signed_windows`, `/proc/sys/net/core/rmem_max` degerlerini okur.
- `calculateLinuxTcpWindowProfile()` kernel formülunu uygular.
- `loadTcpWindowProfile()` platform bazli resolver'dir:
  - Linux: canli sysctl + kernel formulu
  - Windows: ayri Windows resolver branch'i
- `buildTCPSynAlloc()`, `buildTCPAckAlloc()`, `buildTCPDataAlloc()` ayni profile hattini kullanir.

4. Ham entropy toplama
- `std.crypto` kullanilmiyor.
- Linux'ta entropy dogrudan `/dev/urandom` dosyasindan okunuyor.
- Windows'ta `BCryptGenRandom` extern bildirimi ile sistem RNG cagriliyor.
- Ilgili helper: `fillEntropy()`.

5. RFC 8701 GREASE helper'i
- `greaseCodepointFromNibble()` ile `0x0A0A`, `0x1A1A`, ... `0xFAFA` seklinde GREASE codepoint uretiliyor.
- `randomGreaseCodepoint()` ham entropy kullanarak bir GREASE degeri seciliyor.

6. Manual X25519MLKEM768 hybrid key share
- `hybrid_keyshare_len = 1216`
- `mlkem768_share_len = 1184`
- `x25519_share_len = 32`
- `fillHybridKeyShare()` ilk 1184 byte'i ML-KEM-768 bolgesi, son 32 byte'i X25519 bolgesi olarak ham entropy ile dolduruyor.
- Bu asamada odak tam handshake basarisi degil, JA4 yapisal hizalama.

7. MSS siniri icin comptime kontrol
- `tls_client_hello_mss_limit = 1460`
- `maxClientHelloLen()` ile desteklenen maksimum SNI boyuna gore teorik ust sinir hesaplanıyor.
- `comptime` blokta 1460 byte limiti asilirsa derleme hatasi veriliyor.

8. Elle serialize edilen TLS ClientHello
- `buildTLSClientHello()` TLS 1.3 yapisini manuel byte yazimiyla kuruyor.
- `appendInt()` ile big-endian integer serialization yapiliyor.
- Cipher suite, extension, key share, ALPN ve ECH GREASE bolumleri elle siralanmis durumda.

9. Linux interface otomatik tespiti
- `parseDefaultRouteInterface()` `/proc/net/route` ciktisini parse ediyor.
- `detectLinuxDefaultInterface()` varsayilan route interface'ini buluyor.
- `resolveLinuxInterface()` gecersiz interface verilirse default interface'e dusuyor.
- Amac: `eth0` gibi hatali varsayimlari kaldirmak.

10. Testlerde eklenen saf yapisal dogrulamalar
- `tcp_rmem` parser testi
- Linux live sysctl tabanli TCP window profile testi
- SYN builder'in canli kernel TCP window profile ile eslestigi regression testi
- GREASE pattern testi
- Hybrid key share boyut testi
- TLS ClientHello yapisal sayim testi
- Varsayilan route parse testleri
- IP argumaninin interface ile karistirilmamasi testi

11. Standart Zig disi ama proje-ozel kararlar
- `x25519_mlkem768_group = 0x11EC`
- `ech_grease_extension = 0xFE0D`
- `compress_certificate = 27` kullanimi
- Bunlar proje tarafinda JA4/Chrome hizalama hedefi icin manuel secildi.

12. Halen dikkat edilmesi gereken konu
 - Runtime'da `buildTLSClientHelloAlloc(allocator, ...)` donen buffer, ayni allocator ile free edilmelidir.
 - `main` artik TLS buffer'ini `init.gpa` ile ayirip yine `init.gpa` ile free eder.

13. Exact-size TLS allocation ve overflow korumasi
- `tlsClientHelloLen(server_name)` ile ClientHello boyutu onceden tam hesaplanir.
- `buildTLSClientHelloAlloc()` dogrudan o boy kadar `allocator.alloc()` yapar.
- `FixedTlsWriter` icinde her yazimdan once sinir kontrolu vardir.
- Tasma riski gorulurse `@panic("Buffer overflow detected before corruption")` cagrilir.
- Sonunda serialize edilen uzunluk ile beklenen toplam uzunluk esit degilse `@panic("TLS ClientHello serialized length mismatch")` cagrilir.

14. Exact-size SYN allocation ve ownership duzeltmesi
- `buildTCPSynAlloc(allocator, ...)` eklendi.
- `main` artik `syn_packet` icin `init.gpa` kullanir ve ayni allocator ile free eder.
- Boylece onceki `page_allocator` vs `init.gpa` mismatch'i kapatildi.
- SYN serializer artik exact-size packet ayirir ve son uzunlugu kontrol eder.
- TCP header iki byte olarak duzeltildi:
  - byte 12: data offset
  - byte 13: flags
- TCP options 4-byte alignment icin pad edilir.
- Son serialize uzunlugu beklenen boyla uyusmazsa `@panic("SYN packet serialized length mismatch")` cagrilir.
